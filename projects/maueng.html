MauEng

A 3D game engine built from scratch with C++20 and Vulkan.

C++20 Vulkan Engine Graphics CMake In Progress
View on GitHub Watch Demo

Demo

A small demo scene showcasing the engine's rendering and systems.

Renderer

The Vulkan renderer uses a fully bindless, indirect drawing architecture. All geometry is stored in global vertex and index buffers, draw commands are batched with vkCmdDrawIndexedIndirect, and textures live in a descriptor array. A deferred shading pipeline with a depth prepass handles lighting efficiently.

Bindless Rendering

Global buffers with batched indirect draw commands. Textures in a descriptor array - no per-draw binding overhead.

100K Instances

Tested with 100,000 instanced meshes at 60 FPS. Meshes are split into instanced submeshes automatically.

Deferred Shading

G-buffer pipeline with depth prepass to minimize overdraw. Point and directional lighting with configurable exposure.

Asset Pipeline

Assimp integration for all major 3D formats. File-based material loading with default and invalid material fallbacks.

Tone Mapping

Multiple tone mapping modes with adjustable exposure. Custom and automatic exposure controls.

Debug Rendering

Runtime line, sphere, and light visualization API. One-liner debug draws for rapid iteration.

Engine Core

Beyond the renderer, MauEng includes a set of core systems designed for flexibility and ease of use. The event system supports both immediate and queued broadcasts with safe delayed unsubscription. An action-based input system with mapping contexts handles keyboard and mouse input with runtime rebinding.

Event System

Delegates with immediate and queued broadcasts, lambda and member function binding, safe delayed unsubscription.

Input System

Action-based input with mapping contexts, keyboard and mouse support, runtime rebinding.

Component System

EnTT-backed ECS with a wrapper providing the engine's interface for entities, components, and queries.

Logging

Multi-level, multi-category logger with console and file output, configurable rotation and color coding.

Profiling

Built-in profiler outputting to Chrome tracing format with optional Optick integration.

Timer Manager

One-shot and looping timers, next-tick scheduling, pause/resume/reset with handle-based management.
